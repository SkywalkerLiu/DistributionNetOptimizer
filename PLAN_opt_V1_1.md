# 配电线路优化算法重构方案 V1.1

## Summary

本文档用于指导 `DistributionNetOptimizer` 从“连续可行域近似 + 稀疏离散可见图搜索 + 路径后插杆”的实现，升级为“全局连续骨架优化 + 连续接入点优化 + 连续杆位微调”的 V1.1 方案。

V1.1 的核心目标不是简单加密栅格、增加锚点或减小步长，而是把**解空间**从“离散候选点/离散候选边”提升为“连续变量 + 连续代价场 + 分层优化”。地形、坡度、粗糙度和禁区仍可使用栅格表达，但配变位置、主干路径骨架、用户接入点和杆塔位置不再由固定离散候选点唯一决定。

本方案重点解决以下问题：

- 当前路径方向仍受锚点与邻接关系约束，线路方向不够自然。
- 障碍边界附近缺乏连续排斥机制，容易沿障碍边界绕行并围出不合理杆塔。
- 用户接入点依赖离散节点，容易出现“接入方向发散”“挂接点不顺路”。
- 路径后插杆以折线和固定档距为主，杆位缺少连续微调，导致障碍边缘和转折处杆塔形态生硬。
- 当前优化器更像“连续可行域中的离散近似路由器”，而不是严格意义上的全局连续优化器。

V1.1 保持现有输入输出接口总体兼容，不追求一次性引入庞大的统一混合整数模型，而采用**分层连续优化框架**：

1. 上层连续优化：配变位置与主干路径骨架全局搜索。
2. 中层连续优化：主干/分支路径连续平滑与障碍边界排斥修正。
3. 下层离散生成：沿连续路径按弧长自动布杆。
4. 下层连续修正：杆位局部连续微调、净空修复与用户接入修正。
5. 后处理：三相分配、电气校核、局部修复与不可行诊断。

---

## 1. 当前实现问题拆解

### 1.1 现有实现特征

当前版本虽然已经不再使用简单正交规则栅格最短路，但本质仍是以下流程：

1. 在连续可行域中按固定步长抽取锚点。
2. 通过可见图连接有限邻居，得到稀疏几何图。
3. 在图上进行共享低压主干搜索。
4. 路径生成后沿折线按档距自动后插杆。
5. 用户以终端方式挂接到已有节点或已有路径。

该结构相比纯格网最短路已有明显改进，但仍存在“解空间离散化”的根问题。

### 1.2 离散性的主要来源

#### (1) 配变位置搜索仍是采样式

当前配变位置由 `transformer_candidate_step_m` 控制候选点粗采样，再做局部细化。这意味着配变位置仍被限制在一组离散样本上，而不是在连续可行域中直接优化。

#### (2) 路径方向仍由锚点和图边主导

当前路由图由 `path_search_step_m`、边界点抽样、局部种子点和可见图邻接规则共同决定。线路最终方向并不是连续域中自然涌现的，而是从有限锚点和有限连边中“选出来”的。

#### (3) 障碍边界只有“不可穿越”约束，缺少“边界排斥”约束

如果目标函数仅禁止穿越障碍，但不惩罚贴近障碍边界，那么最短路径往往会沿障碍边界滑行。路径后插杆又会沿该折线补杆，最终表现为障碍区周围出现一圈不合理杆位。

#### (4) 用户接入点仍然偏离散

当前用户更接近“挂接到已有离散骨架节点”，而不是“投影到连续主干曲线上的最优接入位置”。这样会使接户方向、分支起点和共享行为受离散节点位置强烈影响。

#### (5) 杆位生成主要是后插，不是连续优化

当前杆塔主要根据折线路径、最大档距和净空修复逻辑自动插入，但缺少对杆位的连续横向/纵向微调，因此在转角、障碍边缘、坡度突变点附近容易出现布局别扭的问题。

## 2. V1.1 重构目标

### 2.1 总体目标

将当前优化器升级为：

**连续可行域中的分层连续优化器，而非由稀疏离散图主导的近似搜索器。**

### 2.2 具体目标

- 配变位置改为连续变量。
- 主干路径骨架改为连续折线/样条控制点优化。
- 障碍约束从“只能不穿越”升级为“不可穿越 + 边界安全裕度惩罚”。
- 用户接入点改为“主干曲线上的连续投影点/连续支接点”。
- 杆位由“路径后插”升级为“沿连续曲线布杆 + 局部连续微调”。
- 可见图、栅格代价场等离散结构只作为初始化工具，不再作为唯一解空间。

### 2.3 不在 V1.1 范围内的内容

V1.1 不要求一次性完成以下高复杂度能力：

- 多台配变联动连续优化。
- 完整的 MILP / MINLP 全局统一建模。
- 精细弧垂力学、耐张杆、转角杆结构选型。
- 完整道路通道、杆型、导线型号联合设计。

V1.1 的重点是先把“路径形态不自然、障碍边围杆、接入点离散化”这几个核心问题解决。

## 3. 核心设计原则

### 3.1 环境可以离散，决策变量必须连续

DTM、坡度、粗糙度、禁区掩膜、基础成本栅格仍可使用栅格表达；但以下决策变量必须提升为连续变量：

- 配变位置 `(x_tx, y_tx)`
- 主干/分支路径的控制点坐标
- 用户接入点在主干曲线上的参数位置
- 杆位在路径曲线上的弧长位置与局部横向偏移

### 3.2 路径优先于杆位

先优化“线路骨架如何走”，再根据该骨架生成和修正杆位。不能把“离散杆位点”反过来当作主路径搜索的核心解空间。

### 3.3 可见图降级为初值生成器

当前可见图与稀疏锚点结构不应直接决定最终方案，而应仅用于：

- 快速生成可行初始骨架；
- 生成候选走廊；
- 为连续优化提供 warm start。

### 3.4 障碍边界需要连续安全裕度

硬禁区仅保证“不能穿越”；实际工程中还需要“不要紧贴边走”。因此必须引入基于障碍距离场的连续排斥项。

### 3.5 用户接入优先共享，但共享位置应连续优化

共享主干的原则保持不变，但用户不再机械挂到离散节点，而应挂到主干连续曲线上的最优支接点。必要时允许引入局部连续 branch point。

## 5. V1.1 算法总流程

### 5.1 新流程概览

```text
Step 1  构建连续可行域与障碍距离场
Step 2  连续优化配变位置
Step 3  生成主干初始走廊（离散结构仅用于初始化）
Step 4  对主干/分支路径执行连续控制点优化
Step 5  用户连续接入点优化
Step 6  沿连续路径按弧长自动布杆
Step 7  杆位局部连续微调
Step 8  三相分配与电气校核
Step 9  局部修复与不可行诊断
Step 10 输出图层、JSON 和可视化
```

### 5.2 与当前版本的关键差异

- 当前版本：先图搜索，再后插杆。
- V1.1：先连续骨架优化，再布杆，再连续微调。

- 当前版本：用户接到离散节点。
- V1.1：用户接到连续曲线上的最佳接入位置。

- 当前版本：障碍只是不允许穿越。
- V1.1：障碍同时产生边界排斥，避免贴边走线。

## 6. 连续目标函数设计

### 6.1 总目标函数

V1.1 推荐总目标函数写为：

```text
J = J_construction
  + J_terrain
  + J_obstacle_margin
  + J_curvature
  + J_attachment
  + J_voltage_drop
  + J_phase_unbalance
  + J_infeasible_penalty
```

其中：

- `J_construction`：线路、杆塔、配变、接户线成本。
- `J_terrain`：坡度、粗糙度、高差、施工困难代价。
- `J_obstacle_margin`：障碍边界安全裕度惩罚。
- `J_curvature`：路径弯折、折线密度、方向突变惩罚。
- `J_attachment`：用户接入长度与接入不顺路惩罚。
- `J_voltage_drop`：压降超限或远距离大负荷接入惩罚。
- `J_phase_unbalance`：共享骨架段的三相累计不平衡惩罚。
- `J_infeasible_penalty`：硬约束冲突的大罚项。

### 6.2 障碍边界排斥项

建议从硬禁区掩膜构建障碍距离场 `D_obs(x, y)`，表示点到最近障碍边界的欧氏距离。

定义安全距离 `d_safe`，则边界排斥项可采用：

```text
phi(d) = ((d_safe - d)_+ / d_safe)^2
```

对路径曲线积分：

```text
J_obstacle_margin = w_obs * ∫ phi(D_obs(gamma(s))) ds
```

说明：

- 若路径距离障碍边界大于 `d_safe`，该项为 0。
- 若路径贴近障碍边界，惩罚快速增大。
- 若路径进入禁区，则由硬约束直接判定不可行。

这项是解决“障碍边围杆”的关键。

### 6.3 路径平滑与转角惩罚

连续路径不能只追求最短，还要惩罚不必要的急转折。可采用：

```text
J_curvature = w_kappa * ∫ kappa(s)^2 ds
```

离散实现时可近似为相邻线段夹角变化平方和：

```text
J_curvature ≈ w_angle * Σ (delta_theta_i)^2
```

### 6.4 用户接入惩罚

用户不应强行接入逆向或严重绕路的主干段。建议定义：

- 接户线长度惩罚。
- 用户方向与主干局部切向方向的不一致惩罚。
- 挂接点过密惩罚（防止所有用户挤在极少数接入点）。

### 6.5 杆位微调惩罚

杆塔微调阶段的局部目标可写为：

```text
J_pole = w_slope * slope(pole)
       + w_obs * phi(D_obs(pole))
       + w_clearance * clearance_risk
       + w_shift * lateral_shift^2
```

## 7. 配变连续位置优化

### 7.1 设计原则

配变选址不再由固定步长候选点主导，而是直接在连续可行域中搜索 `(x_tx, y_tx)`。

### 7.2 推荐算法

V1.1 推荐使用以下任一方案：

1. **粗采样 + 连续局部优化**
   - 先保留当前粗采样作为初筛。
   - 对 Top-K 候选使用连续局部优化（如 Nelder-Mead / Powell）。
   - 优点：改造成本低，适合快速升级。

2. **差分进化 / CMA-ES / 模拟退火**
   - 直接把 `(x, y)` 作为连续变量全局搜索。
   - 每个候选点由下游骨架求解器返回总代价。
   - 优点：更符合“连续全局搜索”目标。

V1.1 推荐先采用方案 1，后续可扩展至方案 2。

### 7.3 评价函数

配变位置评价不仅应包含到用户的距离，还应包含：

- 可建设性；
- 坡度与粗糙度；
- 到主要负荷簇的加权距离；
- 下游骨架可行性与总代价；
- 电压降风险。

即：

```text
score_tx = weighted_distance_to_users
         + terrain_penalty
         + routing_feasibility_penalty
         + voltage_drop_proxy_penalty
```

## 8. 主干走廊初始化与连续路径优化

### 8.1 初始化层：保留离散结构，但仅作 warm start

当前 `_build_routing_graph()` 和可见图搜索逻辑不必立即完全删除，但角色要调整为：

- 为配变到用户簇之间生成初始主干走廊；
- 为主干控制点优化提供初始折线；
- 为不可行区域提供早期诊断。

换言之：

**离散图不再决定最终答案，只负责给连续优化提供起点。**

### 8.2 连续路径表示方式

V1.1 建议使用以下表示之一：

#### 方案 A：折线控制点表示

一条路径表示为：

```text
[T, P1, P2, ..., Pk, End]
```

其中 `P1...Pk` 为连续控制点。

优点：

- 实现简单；
- 容易和现有 `LineString` / GeoPandas 对接；
- 便于分支点、转角点和净空修复点落地。

#### 方案 B：B-spline / 样条表示

更平滑，但会增加实现复杂度。V1.1 建议先用方案 A，在局部平滑阶段再生成样条插值曲线。

### 8.3 连续路径优化变量

对每条主干/分支路径，优化变量包括：

- 中间控制点的 `(x_i, y_i)`
- 必要时控制点数量 `k`
- 分支点在主干路径上的位置参数

### 8.4 连续路径优化目标

每条路径优化时，应联合考虑：

- 长度最小；
- 坡度与粗糙度最小；
- 与障碍边界保持安全裕度；
- 转角更平缓；
- 更利于共享接入；
- 不与现有主干形成不合理平行重复线。

### 8.5 推荐实现策略

#### V1.1 最小可行版本

- 先用当前可见图求初始折线；
- 对折线做 Douglas-Peucker 或类似方式稀疏化；
- 抽取中间关键点作为控制点；
- 在连续域中做若干轮局部扰动优化；
- 每轮评估总目标函数，保留更优结果。

#### 伪代码

```python
init_polyline = build_initial_corridor_via_visibility_graph(...)
control_points = simplify_polyline_to_control_points(init_polyline)

for iter in range(max_iter):
    proposal = perturb_control_points(control_points)
    proposal = project_to_feasible_domain(proposal)
    score = evaluate_continuous_path(proposal, fields, config)
    if score < best_score:
        control_points = proposal
        best_score = score
```

## 9. 用户连续接入点优化

### 9.1 核心变化

用户不再只允许挂接已有离散节点，而允许挂接到主干连续曲线上任意可行位置。

### 9.2 接入模式

V1.1 支持以下三种接入模式：

1. `direct`
   - 用户直接投影到主干/分支曲线上；
   - 若投影距离不超过最大接户线长度，则直接接入。

2. `branch`
   - 若直接投影距离过大，则生成局部短分支；
   - 分支起点在主干连续曲线上优化；
   - 分支路径再做一次简化连续优化。

3. `service_only`
   - 用户与某一连续接入点之间只生成短接户线；
   - 接户线仍然是终端线，不承载其他用户中继。

### 9.3 连续接入计算

给定用户点 `u` 和主干曲线 `gamma(s)`，先求：

```text
s* = argmin ||gamma(s) - u||
```

得到用户在主干上的投影点 `gamma(s*)`。然后根据：

- 投影距离；
- 局部障碍情况；
- 主干切线方向；
- 电压降风险；
- 接入密度；

确定是否直接挂接、改挂其他路径、或生成局部分支。

### 9.4 不顺路惩罚

若用户接入点对应的主干局部切向方向与用户方向夹角过大，则加入惩罚，避免逆向绕接。

## 10. 沿连续路径布杆

### 10.1 布杆原则

杆塔不作为主路径搜索变量，而由连续路径骨架推导出来。基本步骤：

1. 识别固定点：
   - 配变点；
   - 路径起终点；
   - 主要转角点；
   - 分支点；
   - 净空修复强制点。
2. 在固定点之间沿弧长等距布置中间杆塔。
3. 对初始杆位执行连续微调。

### 10.2 弧长布杆

对于一段连续路径，设允许最大平面档距为 `L_max = 50m`，则按弧长分段：

```text
n = ceil(path_length / L_max)
```

再在路径参数空间内按等弧长插值生成杆位，而不是简单沿折线节点或局部直线补点。

### 10.3 转角和分支点加密

在以下位置应允许额外插入杆塔：

- 转角过大处；
- 曲率高处；
- 分支点附近；
- 净空不足处；
- 地形高差突变处；
- 贴近障碍边界但仍可行处。

---

## 11. 杆位局部连续微调

### 11.1 目标

将初始布杆结果从“几何上可用”提升到“形态更自然、施工更合理、净空风险更低”。

### 11.2 可调变量

对于非固定杆位，允许：

- 在路径弧长方向小范围滑动；
- 在路径法向方向小范围横移；
- 必要时重新分配相邻杆间弧长间距。

### 11.3 评价指标

局部微调应综合考虑：

- 杆位处坡度、粗糙度；
- 与障碍边界距离；
- 相邻档距是否超限；
- 导线最小净空；
- 路径形态是否更平顺；
- 横移幅度是否过大。

### 11.4 推荐策略

V1.1 建议使用简单稳定的局部搜索：

- 单杆迭代微调；
- 坐标下降；
- 小步长扰动 + 贪心接受；
- 或小规模 Powell / Nelder-Mead。

固定点（配变点、分支点、关键转角点）不参与或仅允许极小范围修正。

---

## 12. 三相分配与电气校核

### 12.1 三相分配位置不变，但输入结构改变

三相分配仍放在主干骨架、接入关系、杆位和服务线确定之后执行；但 V1.1 中的网络结构来自连续路径与连续接入点，需要在离散化为树结构后再做负荷汇总。

### 12.2 压降评估

压降评估逻辑总体可继承现有实现，但需注意：

- 线路长度应使用连续路径三维长度；
- 用户接入长度应使用连续投影点到用户的真实长度；
- 分支与主干应严格区分；
- 在共享线路段汇总累计负荷。

### 12.3 电气修复优先级

当出现压降或不平衡问题时，优先按以下顺序修复：

1. 相别重分配；
2. 用户接入点局部重选；
3. 局部分支优化；
4. 局部增杆；
5. 局部主干重布；
6. 配变位置重搜索（必要时）。

## 15. 实施阶段建议

### Phase 1：最小可见收益改造

目标：先解决“障碍边围杆”和“贴边走”的核心问题。

实施内容：

1. 增加 `obstacle_distance_field`。
2. 在现有路径评分中引入 `obstacle_margin_penalty`。
3. 用户接入点从“节点挂接”升级为“线段投影 + 节点插入”。
4. 后插杆改为沿折线弧长布杆。

特点：

- 改造成本最低；
- 与现有代码兼容性最好；
- 可以快速改善当前主要视觉问题。

### Phase 2：连续路径优化落地

目标：把可见图从最终解空间降级为初始化器。

实施内容：

1. 新增主干初始走廊生成模块。
2. 对初始折线做连续控制点优化。
3. 在目标函数中正式引入：
   - 障碍边界排斥；
   - 曲率惩罚；
   - 共享接入激励；
   - 电压降代理惩罚。

### Phase 3：连续配变位置优化

目标：将配变位置从离散候选升级为连续变量。

实施内容：

1. 采样式配变候选只保留为初筛。
2. 对 Top-K 候选使用连续优化。
3. 将连续路径代价反馈给配变选址。

### Phase 4：杆位局部连续微调

目标：进一步提高路径和杆位形态自然度、净空与施工合理性。

实施内容：

1. 对初始杆位执行局部微调。
2. 对净空风险点和障碍边缘点重点修正。
3. 输出修正前后对比指标。

## 16. 验收标准

V1.1 至少应满足以下验收指标：

### 16.1 几何形态

- 主干线路不再明显受离散锚点方向束缚。
- 障碍边界周围不应出现成片贴边绕行和围杆现象。
- 转角位置更平顺，折线数减少。
- 杆塔沿路径分布更均匀，不出现明显“抖动式”左右偏摆。

### 16.2 接入行为

- 用户接入点应明显更顺路，避免逆向绕接。
- 空间相近用户更倾向共享主干或共享分支。
- 接户线不应因离散节点问题被迫过长。

### 16.3 电气性能

- 三维长度统计与压降计算正确。
- 最大压降不高于配置阈值，或能明确输出违规诊断。
- 三相不平衡率不高于目标阈值，或能明确输出修复失败原因。

### 16.4 代码结构

- 路径初始化、连续路径优化、连续接入点优化、布杆、杆位微调、校核逻辑模块化分离。
- 不再由单一大函数集中承担所有职责。

## 17. 测试建议

### 17.1 新增单元测试

- 障碍距离场正确性测试。
- 路径边界排斥惩罚测试。
- 用户对主干曲线投影测试。
- 沿弧长布杆测试。
- 杆位微调后不越界、不超档距测试。

### 17.2 新增场景测试

- 单障碍绕行场景：验证路径不过分贴边。
- 双障碍窄通道场景：验证路径居中通过而非贴单侧边缘。
- 多用户同向簇场景：验证共享主干和连续接入。
- 山脊/坡度突变场景：验证杆位微调有效降低局部风险。

### 17.3 回归测试

确保以下现有能力不退化：

- 用户作为终端负荷而非主干中继。
- 低压主干共享行为仍成立。
- 压降、净空、三相分配仍可运行。
- GeoPackage 输出结构保持兼容。

## 18. 对 Codex 的实现要求

### 18.1 必须遵守

- 不要直接删除现有实现后一次性重写全部逻辑。
- 先保留现有流程作为 fallback，再逐步替换为 V1.1 模块。
- 每完成一个阶段，都要保留可运行版本与测试通过状态。
- 所有新增函数必须有清晰 docstring、类型注解和最小单元测试。

### 18.2 推荐策略

- 先在新模块中实现 V1.1 逻辑；
- 再在 `optimizer.py` 中增加 `enable_continuous_optimizer` 分支；
- 保留旧版逻辑便于对照测试；
- 最终再决定是否将旧版降级为 legacy path。

### 18.3 输出要求

Codex 每完成一个阶段后，应至少输出：

1. 修改文件清单；
2. 关键算法说明；
3. 新增配置项；
4. 新增测试项；
5. 与旧版结果对比；
6. 已知限制与后续建议。

## 19. V1.1 一句话结论

V1.1 的核心不是“把离散步长调得更小”，而是：

**把可见图与锚点从最终解空间降级为初始化工具，把配变、路径、接入点和杆位逐步提升为连续变量。先优化线怎么走，再优化杆怎么立。**

这才是当前项目从“离散近似路由器”升级到“全局连续优化器”的正确方向。
