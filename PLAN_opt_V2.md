# 配电线路优化算法重构方案 V2

## Summary

本文档用于指导 `DistributionNetOptimizer` 完全推翻现有“连续可行域近似 + 稀疏离散可见图 + 路径后插杆”的优化框架，重新构建一套面向台区低压配电规划本质问题的新算法。

V2 不再把问题视为“几何路径如何更自然”的路由问题，而是明确将其建模为：

**在满足径向性、压降、容量、净空、障碍绕行和施工约束的前提下，联合优化配变选址、主干-分支拓扑、用户接入关系、三相分配与线路选择，使建设成本最低、整体线损最低、配变与线路段三相负载基本平衡。**

V2 的核心理念是：

1. **先确定工程可行走廊，而不是先在平面上自由找方向。**
2. **再在候选走廊图上优化径向配电树，而不是在连续平面上优化自由曲线。**
3. **相平衡和线损都是正式优化目标，不是后处理附属品。**
4. **杆塔位置不再作为全局主优化变量，而由已选线路边和几何约束自动派生。**

V2 推荐算法名称：

**基于候选走廊图的径向三相配电联合优化算法**

---

## 1. 为什么必须推翻旧算法

### 1.1 旧算法的根问题

现有算法虽然已经摆脱了最粗糙的规则格网正交路由，但本质仍然属于：

- 在平面中搜索几何路径；
- 在几何路径上补杆；
- 再对电气指标做后验校核和局部修正。

这类算法适合解决“路径避障”“形态自然”“折线减少”之类的几何问题，但不适合作为低压台区配电规划的主求解器。

因为台区低压规划的本质并不是“找一条好看的路径”，而是同时决定：

- 配变设在哪里；
- 哪些用户共享哪条主干；
- 哪些分支该独立引出；
- 每个用户接到哪一相；
- 哪些线路段的下游负荷如何累计；
- 压降能否满足；
- 线损是否足够低；
- 配变与各共享线路段的三相是否基本平衡。

也就是说，**真正的核心变量是拓扑和电气分配，而不是几何方向本身。**

### 1.2 旧算法为什么容易失败

旧算法的问题不只是“方向离散”，更深层的原因包括：

1. 把“走线方向”当成主要决策变量，而没有把“主干-分支网络拓扑”作为核心决策对象。
2. 把三相平衡更多地当成后处理或附属评分，而没有把它嵌入网络结构优化的主目标中。
3. 把损耗更多地用距离代理或经验惩罚近似，而没有真实基于线路阻抗和潮流结果去评估总线损。
4. 障碍处理主要依靠路径避障，导致算法容易沿边界擦行，而不是以工程走廊方式稳定绕障。
5. 杆塔被过早纳入主决策空间，使问题维度过高，几何噪声反而掩盖了真正的电气优化目标。

### 1.3 V2 的判断

因此，V2 明确放弃以下旧思路：

- 不再以自由连续路径优化作为主框架；
- 不再以杆塔点集作为主要全局优化对象；
- 不再把可见图或锚点图当作最终解空间；
- 不再通过“路径更顺滑”来替代“网络更优”。

V2 直接回到配电规划问题本身：

**在候选工程走廊上构造最优径向低压配电树。**

---

## 2. V2 的优化目标与约束

## 2.1 优化目标

V2 的总目标分为三部分：

1. **建设成本最低**
2. **整体线损最低**
3. **配变与共享线路段三相负载基本平衡**

推荐总目标函数写为：

```text
min J = C_build + alpha * C_loss + beta * P_unbalance
```

其中：

### (1) 建设成本 `C_build`

包括但不限于：

- 配变固定成本
- 主干线路成本
- 分支线路成本
- 接户线成本
- 杆塔固定成本
- 地形施工附加成本
- 障碍绕行附加成本
- 净空修复附加成本

### (2) 线损成本 `C_loss`

必须基于实际线路电阻与潮流结果计算，而不是仅用几何距离代理。推荐采用：

```text
C_loss = sum_over_edges(I_e^2 * R_e)
```

若需要面向规划期优化，可进一步换算为年损耗电量或折算电费成本。

### (3) 三相不平衡惩罚 `P_unbalance`

应同时约束：

- 配变出口总三相负载平衡；
- 每条共享主干/分支线路段的三相下游累计负载平衡。

推荐写为：

```text
P_unbalance = w_tx * P_tx_unbalance + w_seg * P_segment_unbalance
```

其中：

- `P_tx_unbalance`：配变出口三相不平衡惩罚；
- `P_segment_unbalance`：每条共享线路段的不平衡惩罚；
- 必要时可再加零线电流惩罚。

---

## 2.2 必须满足的硬约束

V2 至少保留并显式建模以下约束：

### 拓扑类约束

- 低压网络必须为**以配变为根的无环径向树**；
- 所有用户必须被接入；
- 用户作为终端负荷节点，不作为其他用户的中继节点；
- 同一共享主干允许承载多个用户的下游负荷；
- 不允许形成无节点闭环；
- 不允许平行重复建设不必要的主干线。

### 电气类约束

- 配变容量不得超限；
- 各线路段压降必须满足要求；
- 各线路段电流/载流能力不得超限；
- 所有单相用户必须分配到 `A/B/C` 三相之一；
- 三相用户必须保持三相接入；
- 必要时设置台区和线路段的不平衡上限作为约束或强惩罚项。

### 几何与施工类约束

- 线路不得穿越树林、水区、人工禁区和不可通行区域；
- 配变、杆塔、接入点不得落入硬禁区或不可建设区域；
- 相邻杆塔档距不得超过上限；
- 接户线长度不得超过上限；
- 导线最小离地净空必须满足要求；
- 杆位应满足坡度、粗糙度和可施工性要求；
- 线路平面投影不应出现无节点交叉。

## 3. V2 的基本布线范式

## 3.1 主干-分支径向树

V2 不采用“任意自由曲线网络”，而采用更符合低压台区工程实践的：

**主干-分支径向树（radial trunk-branch tree）**

其结构特点是：

- 配变作为唯一根节点；
- 若干用户簇通过共享主干分组覆盖；
- 偏离主干的用户通过短分支或接户线接入；
- 网络整体保持单根、无环、可追踪、可统计下游负荷。

### 3.2 为什么不用闭环和自由曲线

对于本项目当前阶段，闭环和网状结构并不适合作为主布线形式，因为：

- 会显著增加约束复杂度；
- 不符合单台区低压方案的主流规划逻辑；
- 会增加运行方式和保护逻辑复杂度；
- 容易偏离当前“成本最低 + 线损最低 + 三相基本平衡”的主目标。

同样，自由连续曲线虽然几何上灵活，但不利于明确表达：

- 哪些线路段是共享主干；
- 各段累计下游负荷是多少；
- 哪些边应进入潮流计算；
- 哪些支路应保留或替换。

因此，V2 采用**候选走廊图 + 径向树选边**作为核心建模方式。

---

## 4. V2 的三层建模框架

V2 将整个问题分为三层：

### 第 1 层：候选走廊生成层

作用：从地形、障碍、可建设区和用户分布中提取一张**工程可行走廊图**。

这一步不求最终最优，只负责回答：

- 哪里原则上允许布线；
- 从哪些通道经过更工程合理；
- 哪些区域只能绕行；
- 哪些节点可以作为主干转折点、分支点、接入点或配变候选点。

### 第 2 层：径向配电树优化层

作用：在候选走廊图上选择：

- 配变位置；
- 线路边集合；
- 用户接入关系；
- 主干与分支结构；
- 共享骨架拓扑。

这一步是 V2 的主优化层。

### 第 3 层：三相分配与潮流评估层

作用：在固定的候选拓扑上，进一步优化：

- 单相用户的 `A/B/C` 相分配；
- 线路段三相负荷累计；
- 配变出口和各线路段的不平衡度；
- 实际线路损耗与电压降。

必要时再把评估结果反馈给上层，触发局部拓扑替换或走廊替换。

---

## 5. 候选走廊图的构建方法

## 5.1 走廊图的定义

走廊图不是栅格，也不是最终线路，而是一张**候选布线图**。

其节点可包括：

- 配变候选点；
- 用户接入候选点；
- 用户簇代表点；
- 走廊交点；
- 必要的转折点；
- 地形通道的关键控制点；
- 人工指定的允许布线锚点。

其边表示：

- 两个节点之间存在一条工程上可接受的候选布线走廊；
- 该边已经满足不穿越硬禁区；
- 该边已具备已知长度、坡度、粗糙度和风险指标；
- 该边可以后续派生为一段或多段实际杆线路径。

## 5.2 走廊图构建原则

走廊图应满足以下原则：

1. **禁区严格剔除**：任何边都不能穿越树林、水区、人工禁区。
2. **边界留安全带**：紧贴禁区边界的边原则上不进入主候选集，或附加更高成本。
3. **优先中轴走廊**：优先选择可建设区域内部的中轴型通道，而不是边界擦行通道。
4. **兼顾共享潜力**：应使多个空间相近用户有机会通过同一主干共享接入。
5. **减少无意义转折**：走廊图本身应避免过密、过碎、过于抖动的边结构。

## 5.3 推荐构建方法

建议组合使用以下方法构建走廊图：

### 方法 A：可建设区骨架提取

从可建设区二值区域中提取中轴骨架或细化中心线，生成天然布线通道。

### 方法 B：用户簇连通通道提取

对用户进行聚类后，在用户簇之间生成若干候选连通走廊。

### 方法 C：障碍膨胀后通道提取

对硬禁区做安全距离膨胀，再在剩余可行域中寻找通道，天然避免贴边走线。

### 方法 D：人工/规则补充

必要时允许添加：

- 道路优先通道；
- 人工指定主干方向；
- 特定位置的强制连接边。

## 6. 配变选址的全新建模

## 6.1 配变选址不再独立几何打分

V2 中，配变位置不应再仅根据“到用户距离 + 坡度粗糙度”单独打分，而应当根据：

**该配变位置作为根节点时，所能形成的最优径向配电树总代价。**

也就是说，配变选址不再是独立几何问题，而是与下层拓扑优化联动的问题。

## 6.2 V2 的配变选址策略

推荐流程如下：

1. 在可建设区内生成一批工程候选点；
2. 结合用户密度、负荷中心、走廊可达性和地形约束做预筛；
3. 取前 `K` 个候选点作为配变根节点备选；
4. 对每个候选点分别求解下层径向树优化；
5. 比较总目标函数，选取最优配变位置。

## 6.3 配变候选点评价的预筛指标

预筛可使用：

- 到各用户簇的加权距离；
- 候选走廊图中的可达性；
- 所在位置的坡度和粗糙度；
- 所在位置与障碍、安全边界的关系；
- 预估主干总长度；
- 预估压降风险。

但最终决定必须由下层真实网络优化结果给出。

## 7. 径向配电树优化层

## 7.1 这一层才是 V2 的核心

在给定一个配变候选点和一张候选走廊图后，V2 的核心任务是：

**从候选边集中选出一棵覆盖全部用户的有根径向树，使建设成本、损耗和不平衡综合最优。**

这一步不再优化“方向”，而是优化：

- 哪些边被选中；
- 哪些边构成主干，哪些边构成分支；
- 各用户如何挂接到共享骨架；
- 哪些用户簇共享同一主干；
- 哪些长支路需要独立引出。

## 7.2 变量建议

在候选图 `G=(V,E)` 上，建议引入如下变量：

### 边选择变量

```text
y_e in {0,1}
```

表示候选边 `e` 是否被建设。

### 配变候选变量

```text
r_k in {0,1}
```

表示配变候选点 `k` 是否被选中。

### 用户接入变量

```text
a_{u,v} in {0,1}
```

表示用户 `u` 是否挂接到接入节点 `v` 或接入边的端点上。

### 连通/流变量

```text
f_e >= 0
```

用于表达该边承载的连通流或累计负荷流，辅助实现：

- 连通性；
- 径向性；
- 下游负荷累计。

## 7.3 径向性约束思路

V2 推荐使用**单商品流或根流约束**来表达“从配变到所有用户的有根径向连通结构”。

常见实现方式：

- 让根节点向外发送总流量；
- 每个用户节点消耗一个单位的连通流或对应负荷流；
- 仅当边被选中时才允许流通过；
- 总边数满足树结构要求；
- 配合连通约束防止出现孤岛和回路。

## 7.4 主干与分支的识别

V2 不要求在 MILP 里一开始就强行定义“主干”与“分支”类别，也可以先由选出的树结构自然形成：

- 累计下游用户较多、承载负荷较大的边可识别为主干；
- 只服务少量末端用户的边可识别为分支；
- 接到单个用户的短边可识别为接户段。

后续在输出图层时再标注 `lv_main`、`lv_branch`、`service_drop`。

## 8. 三相分配与线路负荷累计

## 8.1 三相分配必须成为正式优化层

V2 明确规定：

**单相用户的相别分配不再只是最终附加步骤，而是正式进入优化流程。**

原因在于：

- 相别分配会影响各段线路电流；
- 会影响配变出口三相负载；
- 会影响损耗与电压降；
- 会影响零线电流与线损分布。

## 8.2 相别变量

对于单相用户 `u`，定义：

```text
x_{u,p} in {0,1}, p in {A,B,C}
```

满足：

```text
x_{u,A} + x_{u,B} + x_{u,C} = 1
```

对于三相用户，可固定其为三相接入，不参与单相分配决策。

## 8.3 线路段累计负荷

在固定树结构后，对每一条线路段 `e` 统计其全部下游用户集合 `Down(e)`，从而得到：

```text
S_{e,A}, S_{e,B}, S_{e,C}
```

这一步是后续计算：

- 相电流；
- 三相不平衡；
- 线损；
- 压降；
- 零线电流；

的基础。

## 8.4 不平衡指标

推荐使用：

```text
S_avg = (S_A + S_B + S_C) / 3
unbalance = max(|S_A-S_avg|, |S_B-S_avg|, |S_C-S_avg|) / S_avg
```

并对以下位置分别评估：

- 配变出口；
- 每条共享主干线路段；
- 必要时也可对重要分支段评估。

---

## 9. 线损与压降评估层

## 9.1 为什么必须真实做潮流

V2 不接受仅用“距离越长压降越大、线损越大”的粗代理作为最终评价。原因在于：

- 共享线路段的负荷是累积的；
- 三相分配会改变每段电流；
- 上游边承载的电流远大于末端边；
- 相平衡改善并不等价于总损耗最低；
- 有些拓扑看起来更短，但因负荷集中方式不同，损耗反而更高。

因此，V2 推荐固定拓扑后，对每个候选方案执行**径向三相潮流计算**。

## 9.2 潮流方法建议

对于当前树状低压网络，推荐使用：

**Backward/Forward Sweep（前推回代）径向潮流算法**

因为其具备以下优点：

- 适合树结构；
- 适合高 `R/X` 低压配电网；
- 实现简单；
- 在大量候选方案评估中计算代价可控；
- 可自然统计每条边电流和电压降。

## 9.3 损耗计算

对每一条边 `e`，根据相电流和电阻计算损耗：

```text
P_loss,e = sum_over_phases(I_{e,p}^2 * R_{e,p})
```

总线损：

```text
C_loss = sum_over_edges(P_loss,e)
```

必要时可换算为：

- 日损耗；
- 年损耗；
- 电费损耗成本。

## 9.4 压降计算

每个用户节点电压由其从配变到该点路径上的线路压降累计决定。V2 要求：

- 用户最小电压满足阈值；
- 若不满足，则该方案不可行或触发强惩罚；
- 上层可据此替换拓扑、改接相别或更换配变位置。

## 10. 杆塔与几何生成策略

## 10.1 杆塔不再是全局主优化变量

V2 明确规定：

**全局优化阶段不直接优化每根杆塔坐标。**

原因如下：

- 会使变量数量激增；
- 容易让几何细节掩盖拓扑与电气目标；
- 难以稳定表达共享主干的工程结构；
- 难以在优化过程中实时做可靠潮流评估。

## 10.2 杆塔生成流程

杆塔位置应由已选线路边自动派生，推荐流程如下：

1. 在走廊边几何上生成实际折线或中心线；
2. 识别固定点：配变点、分支点、接入点、重要转折点；
3. 按最大档距沿已选边自动插入中间杆塔；
4. 再做局部净空修复和小范围几何微调；
5. 输出最终杆塔与线路图层。

## 10.3 这样做的好处

- 把“网络拓扑优化”和“杆位几何细化”解耦；
- 先解决真正重要的成本、损耗和三相平衡问题；
- 最终杆位仍可满足档距、净空和障碍避让要求；
- 结果更稳定，且更容易解释。

## 11. 推荐求解框架

V2 不建议用单一算法从头到尾一次性求解。推荐采用：

**候选图精确优化 + 潮流校核 + 大邻域搜索修复**

## 11.1 阶段 A：候选走廊图构建

输入：

- DTM
- 坡度
- 粗糙度
- 可建设区
- 树林/水区/人工禁区
- 用户点

输出：

- 候选走廊节点集 `V`
- 候选走廊边集 `E`
- 每条边的长度、坡度、风险、施工成本等属性

## 11.2 阶段 B：配变候选预筛

输出前 `K` 个配变候选点，供下层评估。

## 11.3 阶段 C：径向树初始优化

对每个配变候选点，在候选走廊图上解一个初始径向树优化问题，目标先以：

- 建设成本
- 损耗代理
- 不平衡代理

为主。

这一阶段可以采用：

- MILP
- 分段线性近似 MILP
- 或 MILP + 启发式近似

## 11.4 阶段 D：相别优化

在固定拓扑上进行单相用户 `A/B/C` 分配优化，可使用：

- MILP
- 局部交换
- 分支级交换
- 贪心 + 改善搜索

## 11.5 阶段 E：三相潮流真实评估

对“拓扑 + 相别”方案做真实潮流评估，计算：

- 总损耗
- 最大压降
- 配变三相负载
- 各段不平衡度

## 11.6 阶段 F：大邻域搜索 ALNS / LNS 外循环

若方案仍可改进，则进行以下高层动作：

- 替换部分主干边；
- 将某个用户簇整体改挂到另一主干；
- 更换局部走廊通道；
- 改变一个分支的接入节点；
- 更换配变候选点；
- 重新做相别交换。

外循环持续到目标函数不再改善或达到迭代次数上限。

## 15. 实施路线

## Phase 1：走廊图替代旧路由器

目标：先彻底摆脱“可见图路径搜索”这一旧核心。

实施内容：

1. 构建候选走廊图模块；
2. 基于障碍膨胀与可建设区中轴生成走廊；
3. 输出候选节点、候选边和属性。

验收：

- 所有候选边均不穿越禁区；
- 走廊图可覆盖全部用户簇；
- 不再出现大量贴障碍边界的候选边。

## Phase 2：径向树优化替代几何路径优化

目标：把主优化器从“路径搜索器”替换为“树选边优化器”。

实施内容：

1. 引入边选择变量和根流/连通流约束；
2. 生成初始径向树；
3. 确保全部用户可接入；
4. 形成共享主干-分支结构。

验收：

- 输出严格无环径向树；
- 用户全部接入；
- 主干共享行为合理；
- 建设成本可统计。

## Phase 3：相别优化与真实潮流校核

目标：把三相分配和线损评估真正嵌入主优化闭环。

实施内容：

1. 新增相别优化模块；
2. 新增三相 BFS 潮流；
3. 计算真实损耗与压降；
4. 将不平衡和压降反馈给上层。

验收：

- 配变和线路段三相负载可统计；
- 线损可真实计算；
- 压降约束可准确校核；
- 违规方案能输出诊断。

## Phase 4：杆塔自动派生与几何修复

目标：将选边结果落到杆塔和线图层。

实施内容：

1. 沿已选边生成实际杆塔；
2. 执行档距与净空修复；
3. 输出最终 `candidate_poles` 与 `planned_lines` 图层。

验收：

- 杆位满足档距约束；
- 净空违规点能修复或诊断；
- 几何输出与树结构一致。

## Phase 5：大邻域搜索外循环

目标：进一步降低成本、线损和不平衡度。

实施内容：

1. 引入主干替换、分支重挂接、走廊替换等高层动作；
2. 反复求解并比较目标函数；
3. 形成多方案比较能力。

验收：

- 相比初始 MILP 解明显改善；
- 结果稳定，不易陷入局部奇异布线。

## 18. V2 的一句话结论

V2 的根本变化不是“把路径做得更连续”，而是：

**把问题从几何路径优化，彻底改造成基于候选走廊图的径向三相配电联合优化。先选工程可行走廊，再选最优径向树，再做相别与潮流，再派生杆塔与几何。**

这才是当前项目真正符合目标函数、符合约束、也更接近工程布线逻辑的优化方向。
